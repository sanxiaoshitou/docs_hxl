# 单点登录（SSO）实现详解

## 什么是单点登录？

单点登录（Single Sign On，简称 SSO）是一种身份验证机制，允许用户使用一组凭证（用户名和密码）登录一次，
就可以访问多个相互信任的应用系统，而无需在每个系统中重复登录。

### 核心优势
- **用户体验提升**：只需登录一次，即可访问所有关联系统
- **安全性增强**：集中管理认证，减少密码泄露风险
- **维护成本降低**：统一的身份管理，简化用户管理流程
- **效率提高**：避免重复登录，提高工作效率

---

## SSO 的核心原理

### 基本概念

1. **认证中心（Authentication Center）**：负责统一处理用户认证
2. **应用系统（Client Application）**：需要接入 SSO 的业务系统
3. **会话（Session）**：用户的登录状态
4. **令牌（Token）**：用于在不同系统间传递用户身份信息

### 工作流程
```
用户 → 应用A → 未登录 → 跳转认证中心 → 登录 → 生成全局会话 → 
返回Ticket → 应用A验证Ticket → 建立局部会话 → 访问成功

用户 → 应用B → 未登录 → 跳转认证中心 → 已有全局会话 → 
返回Ticket → 应用B验证Ticket → 建立局部会话 → 访问成功
```
